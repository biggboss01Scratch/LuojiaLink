# LuoJia Link（珞珈连点）

LuoJia Link 是一个面向武汉大学学生的校园私域社交微信小程序课程项目。项目围绕“校内身份认证、匿名活动发布、广场匹配、双向确认、智能日历和 AI 破冰建议”构建，用于帮助同学在约饭、运动、自习、观影等场景中低成本找到合适的活动伙伴。

当前仓库还是空项目骨架，后续代码计划按前后端分离方式组织：

```text
LuojiaLink/
├── frontend/   # 微信小程序端
├── backend/    # 后端 REST API 服务
├── docs/       # 课程文档、需求与设计说明
└── README.md
```

## 技术栈选型

| 层次 | 建议选型 | 说明 |
| --- | --- | --- |
| 前端 | 微信小程序原生开发 | 作为主要用户入口，适合课程演示、移动端交互和校园场景传播。 |
| 后端 | Spring Boot / 轻量 Web 后端 | 提供 REST API，负责认证、发布、匹配、日历、会话和 AI 服务编排。 |
| 数据库 | MySQL | 存储用户、标签、活动请求、匹配申请、日历时间片、会话、举报和归档数据。 |
| 缓存与状态 | Redis、定时任务、轮询 | P0 阶段可用轮询和定时任务完成过期与通知；后续可用 Redis 承担缓存、会话状态和实时能力。 |
| 外部服务 | WHU-CAS、AI API、商户/地图 API | 通过适配层接入，课程阶段支持模拟认证、模拟地点数据和模板降级。 |
| 接口风格 | REST API | 便于小程序调用、前后端联调和课程验收测试。 |

## 架构设计

项目采用前后端分离架构。微信小程序负责页面展示、表单交互、广场浏览、日历操作和会话入口；后端服务负责业务规则、数据状态、匹配流程和外部服务调用；MySQL 保存核心业务数据；Redis 和定时任务用于处理缓存、请求过期、会话状态和通知类能力。

后端模块可按领域拆分：

- 用户与认证模块：处理微信登录、WHU-CAS 或模拟认证、Token 与用户资料。
- 标签与资料模块：维护兴趣标签、偏好标签和匹配画像。
- 广场发布模块：处理活动请求发布、匿名编号、筛选排序和过期归档。
- 匹配确认模块：处理“想搭”申请、发起者确认、匹配状态流转和临时会话创建。
- 智能日历模块：维护用户空闲时间片，计算共同空闲和冲突状态。
- AI 破冰模块：整合双方标签、活动类别、共同空闲和地点数据，生成建议卡片。
- 外部服务适配层：隔离 WHU-CAS、AI API、商户/地图 API，支持真实实现和 Mock 实现切换。

## 开发阶段

- P0：完成模拟登录、广场发布、想搭申请、发起者确认和匹配结果形成。
- P1：完成日历时间片、共同空闲计算、冲突提示和基础 AI 文本建议。
- P2：接入或模拟接入商户/地图数据，完善 AI 推荐卡片和临时会话体验。
